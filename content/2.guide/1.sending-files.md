# Sending Files

AirShare makes sending files incredibly simple. This guide covers all the different ways you can send files to other devices on your network.

## Prerequisites

Before sending files, make sure:

- ✅ AirShare is running on both devices
- ✅ Both devices are on the **same local network**
- ✅ You can see the recipient in the **Radar** tab
- ✅ You've accepted the EULA

::u-callout
---
icon: i-lucide-info
---
**Tip**: If you don't see the recipient, wait 5-10 seconds for auto-discovery, or check [Troubleshooting Discovery](/troubleshooting/common-issues#discovery-issues).
::

## Method 1: Drag and Drop (Recommended)

The easiest and fastest way to send files:

### Single File

1. Open the **Radar** tab
2. Open your file explorer (Finder/File Explorer)
3. **Drag a file** onto the recipient's card in Radar
4. A confirmation dialog appears showing:
   - File name and size
   - Recipient name
   - Transfer details
5. Click **"Send"**
6. Wait for the recipient to accept

### Multiple Files

You can send multiple files at once:

1. Select multiple files in your file explorer
2. **Drag all files** together onto the recipient's card
3. The confirmation shows all files and total size
4. Click **"Send"**

### Folders

You can also send entire folders:

1. **Drag a folder** onto the recipient's card
2. AirShare will include all files and subfolders
3. Review the total size in the confirmation
4. Click **"Send"**

::u-callout
---
icon: i-lucide-alert-circle
color: amber
---
**File Size Limits**:
- **Free tier**: Maximum 200 MB per transfer
- **Pro tier**: Unlimited file size
::

## Method 2: Click to Send

If you prefer clicking instead of dragging:

1. **Click** on a recipient's card in the Radar
2. A send dialog appears
3. Click **"Select Files"** or **"Select Folder"**
4. Choose files/folder from the file picker
5. Click **"Send"**

## Method 3: System Tray/Menu Bar

Quick sending from the tray icon:

1. **Right-click** the AirShare tray icon
2. Select **"Send Files..."**
3. Choose the recipient from the list
4. Select files to send
5. Confirm and send

## Understanding the Send Confirmation

When you initiate a transfer, you'll see a confirmation dialog:

### Information Displayed

- **Recipient name**: Who will receive the files
- **File list**: Names and sizes of all files
- **Total size**: Combined size of all files
- **Estimated time**: Based on typical network speed

### Actions Available

- **Send**: Proceed with the transfer
- **Cancel**: Abort the transfer
- **Add more files**: Add additional files before sending

## Transfer Process

### Step 1: Waiting for Acceptance

After clicking Send:

1. The recipient receives a notification
2. They see a receive confirmation dialog
3. They can **Accept** or **Decline**
4. You'll see "Waiting for recipient to accept..."

::u-callout
---
icon: i-lucide-clock
color: blue
---
**Timeout**: If the recipient doesn't respond within 60 seconds, the transfer is cancelled automatically.
::

### Step 2: Transfer in Progress

Once accepted:

- **Progress bar**: Shows percentage completed
- **Transfer speed**: Real-time speed in MB/s
- **Time remaining**: Estimated time to completion
- **Files transferred**: Shows which files are being sent

### Step 3: Transfer Complete

When finished:

- ✅ Success notification appears
- Transfer is logged in **History**
- You can send more files immediately

## Advanced Sending Options

### Sending to Multiple Recipients

Currently, you need to send to recipients one at a time:

1. Send to first recipient
2. Wait for transfer to complete
3. Send to next recipient

::u-callout
---
icon: i-lucide-lightbulb
---
**Future feature**: Simultaneous multi-recipient transfers are planned for a future release.
::

### Batch Transfers

For sending many files to the same recipient:

1. **Group files into folders** before transferring
2. Send the entire folder at once
3. This is faster than sending files individually

### Large File Optimization

For very large files (1 GB+):

- ✅ Use **Ethernet** if possible (much faster than Wi-Fi)
- ✅ Connect to **5 GHz Wi-Fi** if wireless (faster than 2.4 GHz)
- ✅ Close other network-intensive apps
- ✅ Don't let your computer sleep during transfer

## Transfer Speeds

Expected speeds based on your connection:

| Connection Type | Typical Speed |
|----------------|---------------|
| Gigabit Ethernet | 100-125 MB/s |
| Wi-Fi 6 (5 GHz) | 50-100 MB/s |
| Wi-Fi 5 (5 GHz) | 20-50 MB/s |
| Wi-Fi 4 (2.4 GHz) | 5-15 MB/s |

::u-callout
---
icon: i-lucide-info
---
**Note**: Actual speeds depend on network congestion, distance from router, and hardware capabilities.
::

## Cancelling a Transfer

You can cancel a transfer at any time:

### During Send

1. Click the **"Cancel"** button in the progress dialog
2. Confirm cancellation
3. The transfer stops immediately
4. The recipient is notified

### Effect of Cancellation

- ❌ No files are transferred (even partially)
- 📝 Transfer is logged as "Cancelled" in History
- ✅ You can start a new transfer immediately

## Common Sending Scenarios

### Scenario 1: Quick Photo Share

**Goal**: Share photos from your phone to your computer

1. Install AirShare on both devices
2. Connect to same Wi-Fi network
3. Open AirShare on both
4. On phone: Select photos and drag to computer in Radar
5. On computer: Accept the transfer
6. Photos appear in Downloads folder

### Scenario 2: Large Video File

**Goal**: Transfer a 5 GB video file

1. **Use Ethernet** on both devices if possible
2. Drag the video file to recipient
3. Click Send
4. Wait for acceptance
5. Monitor progress (should take ~40-80 seconds on Gigabit)
6. ✅ Transfer complete

### Scenario 3: Work Documents

**Goal**: Send project files to colleague

1. Put all files in a folder named "Project Files"
2. Drag the folder to colleague's device in Radar
3. Colleague accepts
4. Entire folder structure transfers intact

## Troubleshooting Sending Issues

### "Recipient not found"

**Problem**: Can't see the recipient in Radar

**Solutions**:
- Ensure both devices are on same network
- Check firewalls allow AirShare
- Wait 10-15 seconds for discovery
- Try restarting AirShare on both devices

### "Transfer failed"

**Problem**: Transfer starts but fails midway

**Solutions**:
- Check network connection is stable
- Ensure both devices stay powered on
- Check recipient has enough disk space
- Try sending smaller batches

### "Transfer timed out"

**Problem**: Recipient didn't accept in time

**Solutions**:
- Notify recipient before sending
- Ask recipient to keep AirShare open
- Try sending again

### "File size exceeds limit"

**Problem**: File is too large for Free tier

**Solutions**:
- Upgrade to Pro for unlimited file sizes
- Split large files into smaller parts
- Compress files before sending

## Best Practices

::u-callout
---
icon: i-lucide-check-circle

### Tips for Successful Transfers

1. **Notify the recipient** before sending large files
2. **Group related files** into folders for organization
3. **Use descriptive names** so recipient knows what they're receiving
4. **Check file sizes** before sending on Free tier
5. **Keep devices connected** to power for large transfers
6. **Use wired connections** when possible for speed
